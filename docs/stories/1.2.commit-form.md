## Story

**As a** user,  
**I want** to select commit type, optional scope, and enter a description,  
**so that** I can define all inputs needed for a conventional commit.

## Acceptance Criteria

1. Select com todos os tipos listados em FR1 (`feat`, `fix`, `docs`, `style`, `refactor`, `perf`, `test`, `build`, `ci`, `chore`, `revert`).
2. Campo escopo opcional com validação (`^[a-z0-9-]*$`).
3. Campo descrição obrigatório (3–72 caracteres).
4. Mensagens de erro acessíveis abaixo dos campos inválidos (`role="alert"`).
5. Layout responsivo mobile e desktop conforme [front-end-spec.md](../front-end-spec.md).
