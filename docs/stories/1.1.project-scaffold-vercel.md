## Story

**As a** developer,  
**I want** a configured React/Vite/TypeScript project with Vercel deployment,  
**so that** the team has a reproducible base to build features.

## Acceptance Criteria

1. Repositório contém app Vite + React + TypeScript + Tailwind configurados.
2. Scripts `dev`, `build`, `lint`, `test`, `typecheck` definidos no `package.json`.
3. `vercel.json` ou configuração equivalente para build estático.
4. Deploy de preview na Vercel funciona a partir do branch `main`.
5. README documenta comandos locais e URL de produção.

## CodeRabbit Integration

> **CodeRabbit Integration**: Disabled  
> Quality validation via manual review and CI (`npm run lint`, `npm test`, `npm run build`).

## Tasks / Subtasks

- [ ] Scaffold Vite React TS na raiz do projeto (AC: 1)
  - [ ] `npm create vite@latest` — template react-ts
  - [ ] Instalar e configurar Tailwind CSS 4
  - [ ] Configurar alias `@/` → `src/` em `vite.config.ts` e `tsconfig`
- [ ] Scripts e quality tools (AC: 2)
  - [ ] ESLint 9 + typescript-eslint
  - [ ] Vitest + jsdom + Testing Library
  - [ ] Scripts: `dev`, `build`, `lint`, `typecheck`, `test`
- [ ] Deploy Vercel (AC: 3, 4)
  - [ ] Criar `vercel.json` (`buildCommand`, `outputDirectory: dist`)
  - [ ] Conectar repo GitHub na Vercel; validar preview em PR
- [ ] CI GitHub Actions (AC: 2, 4)
  - [ ] Workflow `.github/workflows/ci.yml`: ci, lint, typecheck, test, build
- [ ] Documentação (AC: 5)
  - [ ] Atualizar README com setup, scripts e URL Vercel
